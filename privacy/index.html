DopaStop Privacy Policy

    Last updated: 2026-07-06

    DopaStop is a Chrome extension that blurs distracting media and lets you set time limits or downtime schedules for websites. This page explains what data DopaStop handles and how.

    What DopaStop stores

    All data below is stored locally on your own device using Chrome's built-in storage APIs (chrome.storage.sync and chrome.storage.local). Nothing is sent to any server operated by us or anyone else.

    • Settings — your blur preferences, whitelisted sites, site time limits, downtime schedule, and language choice.
    • Usage data — for websites you've chosen to track or limit, DopaStop records the domain name (e.g. "youtube.com") and the number of seconds spent on that site today, so it can enforce your limits and show you the Usage tab. This counter resets automatically every day.

    What DopaStop does not do

    • It does not send any data to an external server. There is no backend — everything runs and stays inside your browser.
    • It does not sell, rent, or share your data with third parties.
    • It does not use your data for advertising or any purpose unrelated to the extension's core blur/limit functionality.
    • It does not record page content, search queries, form input, or anything you type.

    Permissions

    DopaStop requests the following Chrome permissions, used only as described:

    • storage — to save your settings and usage counters locally.
    • alarms — to periodically check downtime schedules even on idle tabs.
    • activeTab + host permissions — to apply the blur effect and enforce limits on whichever page you're viewing, since the extension needs to work on any website you visit.

    Data retention & deletion

    Usage counters reset every day automatically. All stored data (settings and usage) is deleted immediately if you uninstall the extension, or can be cleared any time via Chrome's extension storage settings.

    Changes to this policy

    If this policy changes, the update will be posted on this page with a revised "Last updated" date.
